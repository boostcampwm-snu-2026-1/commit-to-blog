# /executor

## 실행 순서

### 1. 컨텍스트 로드
- `@progress/plan.md` 읽기
- `@progress/current.md` 읽기 (완료 조건 재확인)
- plan.md에 명시된 파일들 확인

plan.md가 없는 경우:
```
plan.md가 없습니다.
→ /planner 를 먼저 실행해 계획을 수립해주세요.
```

### 2. 실행 전 확인

```
─────────────────────────
CP1: 실행 전 확인
─────────────────────────
실행할 단계 (N개):
- [ ] 1. ...
- [ ] 2. ...

변경될 파일:
- ...
─────────────────────────
→ 시작할까요?
```

### 3. 단계별 실행

각 단계를 완료할 때마다:

```
─────────────────────────
[N단계] 완료
─────────────────────────
변경 내용: [요약]
다음: [N+1단계 설명]
─────────────────────────
→ 계속 진행할까요?
```

### 4. 모호한 상황 처리

예상치 못한 상황 발견 시 즉시 멈추고:

```
─────────────────────────
⚠️ 진행 중단 - 확인 필요
─────────────────────────
상황: [무슨 문제인지]

선택지:
A. [방법 A] ← 추천
B. [방법 B]
─────────────────────────
→ 어떻게 할까요?
```

### 5. 전체 완료 후

```
─────────────────────────
CP2: 실행 완료
─────────────────────────
완료된 단계: N/N
변경된 파일:
- ...
─────────────────────────
→ execution-log.md에 기록하고 커밋할까요?
```

execution-log.md 업데이트 후 변경된 모든 파일을 커밋한다:
```
git add [변경된 파일들] progress/execution-log.md
git commit -m "workflow/exec: [task명]"
```

커밋 완료 후:
```
─────────────────────────
커밋 완료
─────────────────────────
→ /evaluator 로 넘어갈까요?
```
