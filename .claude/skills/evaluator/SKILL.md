# /evaluator

## 실행 순서

### 1. 컨텍스트 로드
- `@progress/current.md` 읽기 (완료 조건 확인)
- `@progress/plan.md` 읽기
- `@progress/execution-log.md` 읽기
- `@rules/review.md` 로드

### 2. 완료 조건 검증

current.md의 완료 조건을 하나씩 체크한다:

```
─────────────────────────
CP1: 완료 조건 검증
─────────────────────────
- [x] 조건 1: 통과
- [x] 조건 2: 통과
- [ ] 조건 3: 미통과 → [이유]
─────────────────────────
```

### 3. 코드 품질 체크 (@rules/review.md)

```
─────────────────────────
CP2: 코드 품질 체크
─────────────────────────
- [x] 불필요한 console.log 없음
- [x] 네이밍 규칙 준수
- [ ] [미통과 항목]: [이유]
─────────────────────────
```

### 4. 결과에 따른 분기

**전체 통과 시:**
```
─────────────────────────
검증 통과
─────────────────────────
모든 조건을 충족했습니다.

docs 업데이트 필요 여부:
- [해당 파일]: [업데이트 내용] (있을 경우)

→ done.md로 이동하고 완료 처리할까요?
  (docs 업데이트가 필요하다면 함께 진행합니다)
```

**미통과 항목 있을 시:**
```
─────────────────────────
미통과 항목 있음
─────────────────────────
미통과: [항목명]
원인: [설명]

선택지:
A. /executor 로 돌아가 해당 항목만 재작업
B. 이 상태로 완료 처리 (이유 기록)
C. task를 backlog로 되돌리고 다시 계획
─────────────────────────
→ 어떻게 할까요?
```

### 5. 완료 처리

승인 시:
- `progress/done.md` 에 완료 기록 추가
- `progress/current.md` 초기화
- `progress/plan.md` 초기화
- `progress/execution-log.md` 초기화
- `progress/backlog.md` 에서 해당 항목 완료 처리

파일 정리 후 커밋한다:
```
git add progress/
git commit -m "workflow/eval: [task명]"
```

커밋 완료 후:
```
─────────────────────────
Task 완료 + 커밋 완료
─────────────────────────
[task명] 완료 처리되었습니다.
→ /whattodo-next 로 다음 task를 선택할까요?
```
