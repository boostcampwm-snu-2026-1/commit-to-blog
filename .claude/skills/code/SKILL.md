## 목적

이 스킬은 Claude가 방금 작성한 코드를 사용자가 완전히 이해할 수 있도록 돕기 위한 것입니다. 단순히 "무엇을 하는지"를 나열하는 것이 아니라, "왜 이렇게 구성했는지"와 "각 줄이 전체에서 어떤 역할을 하는지"를 중심으로 설명합니다.

## 설명 순서

### 1단계: 전체 흐름 설명 (간결하게)

코드 전체가 어떤 목적을 위해 어떤 순서로 동작하는지 2~4문장으로 설명합니다.

- 이 코드가 해결하는 문제가 무엇인지
- 데이터나 흐름이 어떻게 이동하는지 (입력 → 처리 → 출력)
- 핵심 구조나 패턴 (예: 콜백, 반복, 조건 분기 등)

비유나 실생활 예시를 활용하면 좋습니다. 기술 용어를 쓸 경우 바로 옆에 짧게 풀어서 설명합니다.

### 2단계: 줄 단위 상세 설명

코드를 위에서 아래로 순서대로 훑으며 각 줄 또는 의미 있는 블록 단위로 설명합니다.

형식 예시:
```
[줄 번호 또는 코드 조각]
→ 설명: 이 줄이 하는 일과 그 이유
```

설명할 때 주의할 점:
- 단순히 코드를 한글로 번역하는 수준에 그치지 말 것 (예: "변수를 선언합니다" ❌)
- 왜 이 방식을 선택했는지, 다르게 쓸 수도 있었지만 이렇게 쓴 이유가 뭔지에 집중 (예: "let 대신 const를 쓴 이유는 이 값이 바뀌지 않기 때문입니다" ✅)
- JavaScript 특유의 동작 방식(호이스팅, 클로저, 비동기 등)이 관련된 경우 간략히 짚어줍니다

### 3단계: 이해 확인 (선택)

설명이 끝나면 사용자가 가장 헷갈릴 수 있는 부분 하나를 골라 간단한 질문을 던집니다.

예: "이 부분에서 `async/await`를 쓴 이유가 무엇인지 생각해봤나요? 힌트: 서버 응답을 기다려야 하는 상황이라서예요."

## 주의사항

- 코드가 없는 경우: "방금 대화에서 작성된 코드를 찾지 못했어요. 설명받고 싶은 코드를 붙여넣어 주세요."라고 안내합니다.
- 코드가 여러 개인 경우: 가장 마지막에 작성된 코드를 기준으로 합니다.
- 길이가 긴 코드: 의미 있는 블록(함수, 루프, 조건문 단위)으로 묶어서 설명합니다.
