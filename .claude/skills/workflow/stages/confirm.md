# 확인 단계

## 목적
리뷰에서 받은 피드백을 반영해 코드를 수정한다. 리뷰 → 확인의 루프는 사용자가 만족할 때까지 반복된다.

## 절차

### 1. 피드백 반영
사용자의 피드백을 바탕으로 코드를 수정한다.

피드백이 모호하다면 바로 수정하지 말고 먼저 확인한다.
예: "로직을 바꿔달라는 건지, 코드 스타일을 바꿔달라는 건지 확인해도 될까요?"

### 2. 수정 내용 요약
수정이 끝나면 어떤 부분을 어떻게 바꿨는지 간단히 요약한다.

```
수정 내용:
- [변경한 부분]: [변경 이유]
```

## 완료 조건
수정 완료 후 → `stages/review.md`로 돌아가 다시 리뷰를 진행한다
