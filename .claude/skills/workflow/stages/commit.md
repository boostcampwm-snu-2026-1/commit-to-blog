# Commit 준비 단계

## 목적
사용자가 커밋을 작성하기 전에 이번 작업에서 무엇이 바뀌었는지 파악할 수 있도록 돕는다.

## 절차

### 1. 변경사항 요약
이번 워크플로우에서 수정/생성된 파일과 핵심 변경 내용을 간단히 정리한다.

```
## 변경사항 요약
- [파일명]: [변경 내용 한 줄 요약]
- [파일명]: [변경 내용 한 줄 요약]
```

### 2. checklist.md 업데이트
해당 항목을 `- [ ]` 에서 `- [x]` 로 변경한다.

### 3. 커밋 안내
"변경사항을 확인하고 커밋 메시지를 작성해주세요." 라고 안내한다.
커밋 메시지는 사용자가 직접 작성한다.
