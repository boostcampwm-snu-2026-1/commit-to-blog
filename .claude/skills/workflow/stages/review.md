# 리뷰 단계

## 목적
사용자가 작성된 코드를 완전히 이해하고 피드백을 줄 수 있도록 코드를 설명한다. 단순히 코드를 전달하는 게 아니라, 왜 이렇게 만들었는지 함께 공유하는 시간이다.

## 절차

### 1. 전체 흐름 설명
작성한 코드 전체가 어떤 목적을 위해 어떤 순서로 동작하는지 2~4문장으로 설명한다.
- 이 코드가 해결하는 문제
- 데이터나 흐름이 어떻게 이동하는지 (입력 → 처리 → 출력)
- 핵심 구조나 패턴

### 2. 줄 단위 설명
코드를 위에서 아래로 순서대로 훑으며 각 줄 또는 의미 있는 블록 단위로 설명한다.

좋은 설명의 기준:
- "변수를 선언합니다" ❌ → 코드를 한글로 번역하는 수준
- "let 대신 const를 쓴 이유는 이 값이 바뀌지 않기 때문입니다" ✅ → 선택의 이유

설계 단계에서 합의했던 내용과 다르게 구현된 부분이 있다면 그 이유도 설명한다.

### 3. 피드백 요청
설명이 끝나면 사용자에게 피드백을 요청한다.

"이 부분에서 수정하고 싶은 게 있거나, 이해가 안 되는 부분이 있으면 말해주세요."

## 완료 조건
- 사용자가 피드백을 주면 → `stages/confirm.md`로 이동
- 사용자가 "좋아", "완료", "없어", "ok" 등 만족 의사를 표현하면 → `stages/commit.md`로 이동
