# /whattodo-next

## 실행 순서

### 1. 컨텍스트 로드
- `@progress/backlog.md` 읽기
- `@progress/done.md` 읽기 (완료된 항목 파악)
- `@progress/current.md` 읽기 (진행 중인 게 있는지 확인)

진행 중인 task가 있다면:
```
─────────────────────────
현재 진행 중인 task가 있습니다: [task명]
─────────────────────────
→ 이 task를 계속할까요, 아니면 새 task를 선택할까요?
```

### 2. Backlog 분석 후 후보 제시

아래 기준으로 분석한다:
- 우선순위
- 의존성 (선행 task 완료 여부)
- 현재 블로커 여부

```
─────────────────────────
CP1: Task 선택
─────────────────────────
추천 후보:

1. [task명] ← 추천
   이유: [우선순위 높음 / 의존성 없음 등]

2. [task명]
   이유: [...]

3. [task명]
   이유: [...]
─────────────────────────
→ 어떤 task를 진행할까요? (번호 또는 직접 입력)
```

### 3. 완료 조건 정의

선택 후, 완료 기준 초안을 작성해 확인받는다:

```
─────────────────────────
CP2: 완료 조건 확인
─────────────────────────
[task명]의 완료 조건 초안:
- [ ] ...
- [ ] ...
─────────────────────────
→ 이 완료 조건으로 맞나요? 추가하거나 바꿀 게 있나요?
```

### 4. 추가 컨텍스트 확인

```
─────────────────────────
CP3: 제약사항 확인
─────────────────────────
→ 이번 작업에서 특별히 신경 쓸 제약사항이나 참고할 게 있나요?
  (없으면 바로 저장합니다)
```

### 5. current.md 저장

확인 완료 후:

```
─────────────────────────
CP4: 최종 확인
─────────────────────────
[정리된 내용 요약]
─────────────────────────
→ progress/current.md 에 저장할까요?
```

저장 후 backlog.md에서 해당 항목을 진행 중으로 표시한다.

저장 완료 후 커밋을 진행한다:
```
git add progress/current.md progress/backlog.md
git commit -m "workflow/task: [task명]"
```

커밋 완료 후:
```
─────────────────────────
커밋 완료
─────────────────────────
→ /planner 로 넘어갈까요?
```
