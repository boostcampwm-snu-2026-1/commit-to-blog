# /summary

## 실행 순서

### 1. 데이터 수집
- `@progress/done.md` 읽기 → 가장 최근 완료된 task 확인
- `git log --oneline` 실행 → 이번 사이클 커밋 목록 확인
  (workflow/task ~ workflow/eval 범위)
- 이번 사이클에서 생성/변경된 docs 파일 확인

### 2. 요약 출력

아래 형식으로 출력한다:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━
사이클 요약
━━━━━━━━━━━━━━━━━━━━━━━━━━━
Task: [task명]
완료일: [날짜]

## 한 줄 요약
[이 사이클에서 뭘 했는지 한 문장]

## 주요 결정
- [핵심 결정 1]
- [핵심 결정 2]

## 생성/변경된 파일
- [파일명]: [한 줄 설명]

## 다음 사이클 힌트
backlog에서 다음 우선순위 task: [task명]
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 원칙
- 길게 쓰지 않는다. 핵심만 뽑는다.
- "무엇을 했나" 보다 "왜 그렇게 결정했나"를 강조한다.
- 다음 사이클에 참고할 만한 내용이 있으면 힌트로 남긴다.
