# commit-msg

커밋 메시지를 자동으로 작성한다.

## 실행 순서

1. `git status`를 실행해 변경된 파일 목록을 보여준다.

2. 사용자에게 전체를 stage할지 확인한다. 확인되면 `git add .`를 실행한다.

3. `git diff --staged`로 staged된 변경사항을 분석한다.

4. `git log --oneline -3`으로 최근 커밋을 확인해 다음 순번을 파악한다.

5. `CLAUDE.md`의 커밋 컨벤션을 따라 아래 형식으로 메시지를 작성한다:

```
<type>: #<순번> <제목>

- 확인내용: <변경사항 기반으로 실제 확인된 내용 작성>
- 이해 안 됐던 부분: <없음 또는 해당 내용>
```

6. 작성한 커밋 메시지를 출력한다. 사용자가 커밋 메시지를 확인/수정하고 직접 커밋할 예정이므로 커밋은 직접 실행하지 않는다.
