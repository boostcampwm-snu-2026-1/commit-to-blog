# AI-Blog Project Skills

이 폴더의 스킬은 `Mission.md`와 `checklist.md`를 기준으로 AI-Blog 미션을 반복 가능하게 진행하기 위한 프로젝트 로컬 지침이다.

## Skills

- `skill_creator.md`: 프로젝트 로컬 스킬을 새로 만들거나 수정할 때 사용한다.
- `project-structure-architect.md`: React/Express 파일 및 디렉토리 구조를 설계하거나 정리할 때 사용한다.
- `github-api-integrator.md`: GitHub Repository, Branch, Commit, diff 연동을 Express 서버에서 구현하거나 검토할 때 사용한다.
- `llm-draft-generator.md`: 선택한 커밋 데이터를 LLM 블로그 초안으로 바꾸는 서버 흐름, 프롬프트, 응답 처리를 다룰 때 사용한다.
- `smart-blog-ui-builder.md`: Repository 선택부터 초안 편집, 저장된 포스트 카드까지 React UI를 만들거나 검토할 때 사용한다.
- `mission-verifier.md`: 주차별 계획, 구현물, 스킬 묶음을 `Mission.md`, `checklist.md`, 보안 규칙, 검증 기준에 맞춰 재검토할 때 사용한다.

## Usage Rule

작업을 시작할 때 현재 요청에 가장 좁게 맞는 스킬 하나를 먼저 고른다. 여러 영역이 동시에 걸리면 `mission-verifier.md`로 범위와 검증 기준을 잡고, 필요한 세부 스킬만 추가로 읽는다.
