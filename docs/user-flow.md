# 사용자 흐름 (User Flow)

> 화면 단위로 "사용자가 무엇을 보고, 무엇을 클릭하고, 어떤 상태가 변하는가" 를 정리한다.
> PDF 예시 화면 2종(저장된 포스트 / 포스트 작성) 을 기준으로 함.

---

## 화면 지도

```
┌─────────────────────────┐
│   Saved Posts (홈)      │ ◀────────────┐
│   /                     │              │
│                         │              │
│  [+ 블로그 생성] ───────┼─▶ Create     │
│  카드 1 [수정] [발행]   │   /create    │
│  카드 2 [수정] [발행]   │              │
│  카드 3 …               │              │
└─────────────────────────┘              │
                                          │
┌─────────────────────────┐               │
│   Create Post           │               │
│   /create               │               │
│                         │               │
│   [저장소 입력]         │               │
│   [브랜치 선택]         │               │
│   [최근 커밋 카드]      │               │
│   [AI 요약 패널]        │               │
│   [편집기]              │               │
│   [블로그로 저장 및 게시] ─────────────┘
└─────────────────────────┘
```

---

## Flow A — 새 포스트 만들기 (Happy Path)

1. 사용자가 `/` 진입 → 저장된 포스트 카드 목록을 본다.
2. `[+ 블로그 생성]` 버튼 클릭 → `/create` 진입.
3. 저장소 검색창에 입력 → 디바운스 후 `GET /api/repos?q=...` 호출.
   - 검색 결과 드롭다운에서 저장소 선택.
4. 브랜치 드롭다운이 활성화 → `GET /api/repos/:owner/:repo/branches` 호출.
   - 기본값은 `default branch` (보통 `main`).
5. 최근 커밋 카드가 나타남 → `GET /api/repos/:owner/:repo/commits?branch=...` 호출.
6. 사용자가 커밋 카드 하나(또는 여럿)를 선택.
7. `[요약 생성]` 클릭 → 우측 패널에 로딩 표시 → `POST /api/posts/draft` 호출.
   - 응답으로 받은 초안이 편집기 영역에 채워짐.
8. 사용자가 편집기에서 수정.
9. `[블로그 포스트로 저장 및 게시]` 클릭 → `POST /api/posts` 호출 → `/` 로 돌아감.

## Flow B — 저장된 포스트 다시 편집

1. `/` 에서 카드의 `[수정하기]` 클릭 → `/posts/:id/edit` 진입.
2. `GET /api/posts/:id` 로 본문 로드 → 편집기에 표시.
3. 수정 후 `[저장]` 클릭 → `PUT /api/posts/:id` 호출 → `/` 로 돌아감.

## Flow C — 발행

1. `/` 에서 카드의 `[발행하기]` 클릭.
2. 확인 모달 (선택사항) → 확정 시 `PATCH /api/posts/:id/publish` 호출.
3. 카드의 상태 뱃지가 `draft` → `published` 로 바뀜.

## 에러 시나리오

| 시나리오 | 사용자에게 보이는 것 | 회복 경로 |
|---|---|---|
| GitHub 토큰 누락 / 만료 | 상단 배너 "GitHub 인증이 필요합니다. `.env` 확인" | 토큰 갱신 후 새로고침 |
| GitHub Rate Limit (403) | 토스트 "잠시 후 다시 시도해주세요" + 남은 시간 표시 | 자동 재시도 (지수 백오프) |
| OpenAI 호출 실패 | AI 요약 패널에 에러 박스 + `[다시 시도]` 버튼 | 같은 커밋으로 재요청 |
| diff 가 너무 큼 (token 초과) | "선택한 커밋의 변경량이 너무 큽니다. 커밋을 줄이거나 파일 일부만 선택해주세요." | 커밋 수 줄이기 |
| 저장 실패 (서버 다운) | "저장하지 못했습니다. 잠시 후 다시 시도해주세요." | 자동 재시도 1회 |

## 핵심 인터랙션 디테일

- **검색 입력 디바운스**: 300ms
- **AI 요약 생성 중**: 입력 영역 disable, 스피너 + "AI가 분석 중입니다…" 텍스트
- **카드 호버**: 살짝 떠오르는 그림자 (Tailwind `hover:shadow-md`)
- **발행 버튼**: 미저장 변경사항이 있으면 disable
- **편집기 글자 수 표시**: 우하단 `0 chars` 카운터 (PDF 예시 화면 그대로)
