# 주요 인터랙션 정의

사용자 관점에서 각 화면의 핵심 동작을 정의한다.

---

## 위저드 — 블로그 생성 흐름 (`/new`)

### Step 1 — PAT 입력
- 입력창은 비밀번호 타입 (기본 가려짐)
- 확인 버튼 클릭 → 로딩 스피너 + 버튼 비활성화
- 성공 → Step 2로 이동
- 실패 → "유효하지 않은 PAT입니다" 에러 메시지 인라인 표시

### Step 2 — 저장소 선택
- 진입 시 자동 로드 (스켈레톤)
- 검색으로 실시간 필터링
- 저장소 클릭 → 하이라이트 → 다음 버튼 활성화
- 로드 실패 → 에러 + 재시도 버튼

### Step 3 — 브랜치 선택
- 진입 시 자동 로드
- 클릭 → 선택됨 → 다음 버튼 활성화

### Step 4 — 커밋 선택
- 진입 시 자동 로드
- 체크박스 다중 선택, 최소 1개 선택해야 다음 버튼 활성화
- 선택 개수 표시 ("3개 선택됨")

### Step 5 — AI 생성 중
- 진입 시 자동 실행 (diff 수집 → Gemini 호출)
- 단계별 상태 텍스트 표시 ("변경사항 분석 중…" → "초안 작성 중…")
- 실패 → 에러 메시지 + 다시 시도 버튼 (Step 4로 돌아가지 않고 재시도)

### Step 6 — 편집기
- 제목: AI 응답 첫 `# ` 줄 자동 파싱해서 채움 (수정 가능)
- 마크다운 편집 + 실시간 미리보기
- 저장 버튼 → 로딩 중 비활성화
- 성공 → Step 7
- 실패 → 에러 토스트
- 뒤로 가기 → "작성 중인 내용이 사라집니다" 경고 후 이동

### Step 7 — 저장 완료
- "포스트 편집" → `/posts/[id]`
- "목록으로" → `/posts`
- "새 글 작성" → 상태 초기화 후 Step 1

---

## 포스트 목록 (`/posts`)
- 마운트 시 자동 로드, 로딩 중 스켈레톤 카드 표시
- 포스트 없을 때 → "아직 작성된 글이 없습니다 + 새 글 작성 버튼" Empty State
- 수정하기 → `/posts/[id]`
- 발행하기 클릭 → 즉시 토글, 버튼 텍스트 "초안으로 전환" ↔ "발행하기" 변경
- 발행 실패 → 에러 토스트 + 이전 상태로 롤백

---

## 포스트 편집 (`/posts/[id]`)
- 마운트 시 자동 로드
- 제목 / 내용 수정 후 저장 → 성공 토스트
- 발행 / 초안 토글
- 삭제 버튼 → "정말 삭제하시겠습니까?" 확인 후 삭제 → `/posts`로 이동
