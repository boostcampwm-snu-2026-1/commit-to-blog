# 1주차 요구사항/범위 확정

## 기준 문서
- `docs/assignment_requirements.md`
- `docs/requirements.md`
- `AGENTS.md`

## 1) 필수 기능 범위 (MVP)
- GitHub 저장소 목록 조회
- 선택 저장소 커밋 목록 조회
- 선택 커밋 diff 조회
- AI 인터뷰 질문 생성(1~2개)
- 인터뷰 액션: 답변 제출 / 힌트 보기 / 모르겠어요(해설 보기) / 질문 스킵
- 인터뷰 결과 기반 마크다운 초안 자동 생성(답변 성공/모르겠어요/스킵 분기)
- 마크다운 편집 + 프리뷰
- 포스트 저장
- 저장 포스트 목록 조회
- 저장 포스트 재편집

## 2) MVP 제외 범위
- 실서비스 배포/운영 자동화(CI/CD)
- 다중 사용자 인증/권한 관리(고급)
- 실시간 협업 편집
- 고급 분석 대시보드(학습 통계)
- 외부 블로그 플랫폼 자동 발행 연동

## 3) 기능 우선순위
- P0: 저장소/커밋 선택, diff 조회, AI 질문 생성, 인터뷰 4액션, 분기별 초안 생성, 편집/프리뷰, 저장/목록/재편집
- P1: 인터뷰 품질 개선(프롬프트 고도화), 예외 처리 UX 강화, 기본 관측 로깅 정교화
- P2: 검색/필터 고도화, 통계/리포트, 발행 채널 확장

## 범위 확정 메모
- 외부 API 호출은 서버(Express)에서만 수행한다.
- 민감정보(`GITHUB_TOKEN`, `GEMINI_API_KEY`)는 서버 환경변수로만 관리한다.
- 데이터 저장소는 MongoDB(Mongoose) 기준으로 설계한다.
