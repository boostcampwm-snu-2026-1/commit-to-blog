# 보안/테스트 기준 (Week 1)

## 1) 토큰/시크릿 관리 정책

적용 범위:
- `GITHUB_TOKEN`
- `GEMINI_API_KEY`
- 추후 인증 도입 시 `JWT_SECRET` 등

정책:
1. 시크릿은 서버 런타임 환경변수에서만 로드한다.
2. 클라이언트 번들(`apps/client`)에 시크릿 값이 포함되면 안 된다.
3. `.env` 파일은 Git에 커밋하지 않는다 (`.gitignore` 필수).
4. 저장소에는 `.env.example`만 두고 실제 값은 비워둔다.
5. 로그/에러 응답에 토큰 원문을 출력하지 않는다(마스킹 처리).
6. 외부 API 호출은 반드시 `apps/server`에서만 수행한다.
7. 운영/개발 키를 분리하고, 노출 의심 시 즉시 폐기/재발급한다.

권장 환경변수 목록:
- `NODE_ENV`
- `PORT`
- `MONGODB_URI`
- `GITHUB_TOKEN`
- `GEMINI_API_KEY`
- `LOG_LEVEL`

마스킹 규칙:
- 토큰/키는 앞 4자리만 남기고 `****` 처리
- 에러 객체 직렬화 시 `headers.authorization`/`x-api-key` 제거

## 2) 예외 시나리오 처리 기준

### A. 빈 diff
- 조건: 변경 파일 없음 또는 patch 미포함
- 처리:
1. 인터뷰 질문 생성 대신 "요약 모드"로 자동 전환
2. 사용자에게 "학습 질문 생성이 어려워 커밋 요약으로 전환" 안내
3. 상태값 `SKIPPED_EMPTY_DIFF` 기록

### B. GitHub API 실패
- 조건: 4xx/5xx, rate limit, 네트워크 실패
- 처리:
1. 서버는 `GITHUB_API_ERROR` 반환
2. 429(rate limit)은 재시도 대기 시간과 함께 안내
3. 동일 요청 재시도 최대 2회(지수 백오프)

### C. LLM API 실패
- 조건: 모델 오류, 안전 필터 차단, 형식 불일치
- 처리:
1. 1차 재시도 1회
2. JSON 스키마 검증 실패 시 "스키마 강제 재요청" 프롬프트로 1회 추가 재시도
3. 최종 실패 시 `LLM_API_ERROR` 반환 + 수동 스킵 유도

### D. 타임아웃
- 기준:
- GitHub API: 10초
- LLM API: 20초
- DB 쿼리: 5초

- 처리:
1. 타임아웃 발생 시 `*_TIMEOUT` 에러코드 반환
2. 사용자 UI에는 "재시도" 버튼 제공
3. 서버 로그에는 요청 ID와 소요 시간만 기록

### E. 과대 diff
- 조건: 파일 수/라인 수가 임계치 초과
- 기준(초안):
- 파일 수 30개 초과 또는 patch 15,000자 초과

- 처리:
1. diff 요약 전처리(핵심 파일 상위 N개)
2. 질문 수 1개로 축소
3. 안내 문구: "변경량이 커 핵심 변경 중심으로 질문합니다"

## 3) 완료 판정용 테스트 체크포인트

### P0 플로우 테스트
1. 저장소 선택 -> 커밋 선택 -> diff 조회 성공
2. 인터뷰 시작 -> 질문 생성 -> 답변 제출 -> 피드백 수신
3. 초안 생성 -> 수정 -> 저장 -> 포스트 목록 조회

### 예외 테스트
1. 빈 diff 커밋 선택 시 요약 모드 자동 전환
2. GitHub API 실패 시 표준 에러 포맷 반환
3. LLM 스키마 불일치 시 재시도 후 실패 처리
4. 타임아웃 발생 시 재시도 UX 노출
5. 스킵 선택 시 릴리즈노트형 초안 생성

### 보안 테스트
1. 클라이언트 번들에 `GITHUB_TOKEN`, `GEMINI_API_KEY` 문자열 미포함
2. 서버 로그에 토큰 원문 노출 없음
3. API 응답 payload에 민감정보 필드 없음
4. `.env`가 Git 추적 대상이 아님

### 데이터/무결성 테스트
1. `interviewSessions(userId, createdAt)` 인덱스 동작 확인
2. `interviewTurns(sessionId, turnIndex unique)` 중복 방지 확인
3. `posts(userId, updatedAt)` 최신순 조회 확인

### 완료 기준
- P0 플로우 3개 전부 통과
- 예외 테스트 5개 중 blocker 0건
- 보안 테스트 4개 전부 통과
- 핵심 API 응답이 계약서(`docs/api_contract.md`)와 일치
