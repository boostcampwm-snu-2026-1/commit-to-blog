# AI Workflow

## 목적
분석, 설계, 구현, 검증을 한 번에 밀어붙이기보다 반복 가능한 흐름으로 관리한다.

## 현재 Workflow
1. 요구사항을 사용자 흐름과 데이터 흐름으로 다시 쪼갠다.
2. 한 번에 끝낼 수 있는 수직 슬라이스를 정한다.
3. 외부 API 가 없어도 검증 가능한 데모 폴백을 먼저 둔다.
4. 구현 중 반복되는 판단 기준은 Skill 로 추출한다.
5. 변경 후에는 타입체크와 사용자 시나리오 기준으로 검증한다.
6. 막힌 지점과 개선점을 다음 주차 계획에 반영한다.

## 이번 프로젝트에서 AI 를 쓰는 방식
- 기능 요구사항을 분석해 누락된 데이터 구조를 찾는다.
- GitHub commit detail 을 LLM 입력 형식으로 정제한다.
- 개발 블로그 초안의 섹션 구조를 일관되게 유지한다.
- 문서, 코드, 검증 항목이 서로 어긋나지 않는지 점검한다.

## 이번에 만든 Skill 사용 방식
- `skills/commit-to-blog-loop/SKILL.md`
- 새 기능을 추가할 때 아래 순서를 반복한다.
  - 범위 고정
  - API/데이터 계약 정의
  - 데모 폴백 확보
  - UI 와 저장 흐름 연결
  - 검증 항목 갱신
