# 1주차 계획

## 목표
- 요구사항을 기능 단위로 분해하고 MVP 범위를 고정한다.
- GitHub API, LLM API, 저장 구조를 연결하는 기본 아키텍처를 확정한다.
- AI Workflow 와 커스텀 Skill 을 정의해 이후 개발 속도를 높인다.

## 이번 주 핵심 산출물
- 서비스 요구사항 분석 문서
- 데이터 구조 및 상태 흐름 설계 문서
- 커스텀 Skill 초안
- React + Express 모노레포 초기 세팅
- GitHub 목록/브랜치/커밋 조회 API
- 저장된 포스트 목록 조회용 DB 스키마

## 작업 항목
1. 기능 요구사항을 사용자 흐름 기준으로 다시 정리한다.
2. 화면 기준으로 필요한 데이터 구조를 도출한다.
3. 서버와 클라이언트 디렉토리 구조를 설계한다.
4. GitHub API 연동 방식을 Express 서버 중심으로 설계한다.
5. OpenAI 연동 프롬프트 구조와 실패 시 폴백 전략을 정한다.
6. SQLite 저장 스키마를 정하고 초안 테이블을 만든다.
7. 반복 작업을 줄일 수 있는 프로젝트 전용 Skill 을 만든다.

## 완료 조건
- `docs/architecture.md` 에 기술 선택 이유, 범위, 데이터 흐름이 정리되어 있다.
- `skills/commit-to-blog-loop/SKILL.md` 가 생성되어 있다.
- 저장소에서 `npm run check` 가 통과한다.
- GitHub/AI 토큰이 없어도 데모 모드로 화면 검증이 가능하다.

## AI 활용 포인트
- 요구사항 분석을 표준화하기 위해 Skill 기반 체크리스트를 만든다.
- API 응답 스키마와 UI 상태를 AI에게 반복 점검받는다.
- 설계와 구현 사이에서 누락되는 항목을 AI Workflow 문서로 추적한다.
