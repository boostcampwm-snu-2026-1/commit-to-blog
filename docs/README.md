# Knowledge Base

이 `docs/` 디렉터리는 **프로젝트 코드 구조를 미러링**한다.
특정 코드 작업 전, 해당 경로의 md 파일을 참고한다.

## 탐색 방법

| 작업 종류 | 참고 경로 |
|----------|----------|
| 프론트엔드 | `docs/frontend/` |
| 백엔드 | `docs/backend/` |
| 기술 의사결정 배경 | `docs/adr/` |

## 파일 작성 규칙
- 코드 디렉터리가 생기면 대응하는 docs 파일도 함께 만든다
- **"왜"** 중심으로 작성한다 (what은 코드가 설명한다)
- 결정이 바뀌면 `docs/adr/` 에 기록한다
