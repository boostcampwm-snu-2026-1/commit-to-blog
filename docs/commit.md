# Commit Message Guide

아래 형식으로 작성하고, PowerShell에서는 백틱(`) 줄바꿈을 사용한다.

예시:

```powershell
git commit -m "feat: #번호 작업 제목" -m "- 변경내용: 이번 작업에서 실제로 변경/구현한 핵심 내용" -m "- 참고내용: 아직 확신 없는 부분, 추가 확인이 필요한 사항"
```

## Type 결정 기준

- `feat`: 사용자 관점의 새로운 기능 추가
- `fix`: 버그 수정, 오작동/회귀 해결
- `docs`: 문서 추가/수정(코드 동작 변화 없음)
- `refactor`: 기능 변화 없이 구조 개선
- `test`: 테스트 코드 추가/수정
- `chore`: 빌드/설정/도구/의존성 등 잡무성 변경

판단 규칙:
- 코드 기능 변경이 있으면 `feat`/`fix`를 우선한다.
- 문서만 바뀌면 `docs`를 사용한다.
- 환경설정만 바뀌면 `chore`를 사용한다.
- 문서 + 설정처럼 섞인 경우, 사용자 영향이 큰 변경 기준으로 대표 타입 1개를 고른다.

## 이번 변경 권장 타입

현재 변경은 문서 작성/수정이 중심이므로 `docs` 권장.

```powershell
git commit -m "docs: 아키텍처 문서화 및 체크리스트/인코딩 정리" `
  -m "- 변경내용: docs/architecture_data_design.md 작성(시스템 흐름, 상태 머신, 데이터 설계 포함)" `
  -m "- 변경내용: docs/checklist.md 아키텍처/데이터 설계 항목 완료 처리 및 한글 깨짐 복구" `
  -m "- 참고내용: PowerShell UTF-8 기본 인코딩 설정 적용으로 문서 인코딩 재발 방지"
```
