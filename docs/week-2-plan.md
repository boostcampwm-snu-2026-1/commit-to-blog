# 2주차 계획

## 목표
- 실제 사용 흐름이 끊기지 않도록 생성, 편집, 저장, 발행까지 연결한다.
- 검증 기준을 명확히 두고 데모 모드와 실연동 모드를 모두 점검한다.
- 1주차에 만든 Skill 과 Workflow 를 보완해 다음 반복 개발에 재사용 가능하게 만든다.

## 이번 주 핵심 산출물
- 블로그 생성 화면 완성
- 저장된 포스트 카드 목록 및 상세 편집 화면 완성
- OpenAI 기반 초안 생성 기능 연결
- 발행 상태 업데이트 기능 구현
- README 실행 가이드 및 검증 시나리오 정리

## 작업 항목
1. 커밋 선택 후 블로그 초안 생성 API 를 완성한다.
2. 마크다운 편집과 미리보기 인터페이스를 완성한다.
3. 저장된 포스트 목록 카드와 상세 편집 화면을 완성한다.
4. 발행 처리와 상태 표시를 추가한다.
5. `.env` 기반의 비밀키 관리 방식을 README 에 정리한다.
6. 실연동 실패 시에도 데모 플로우가 유지되는지 점검한다.
7. Skill 에서 유용했던 절차와 부족했던 절차를 반영해 개선한다.

## 완료 조건
- 사용자는 저장소 선택부터 포스트 저장까지 한 번에 수행할 수 있다.
- 저장된 글을 다시 열어 수정하고 발행할 수 있다.
- GitHub/LLM 실연동 시 정상 동작하고, 토큰이 없을 때는 데모 모드로 대체된다.
- 검증 절차가 `docs/ai-workflow.md` 와 README 에 문서화되어 있다.

## 리스크와 대응
- GitHub diff 데이터가 너무 길어질 수 있다.
  - 파일별 patch 길이를 제한하고 핵심 메타데이터를 우선 전달한다.
- API 키 부재로 시연이 어려울 수 있다.
  - 데모 데이터와 로컬 초안 생성 폴백을 함께 둔다.
- 2주 범위에서 OAuth 까지 구현하면 범위가 커질 수 있다.
  - 현재는 서버 `.env` 기반의 Personal Access Token 방식으로 고정한다.
