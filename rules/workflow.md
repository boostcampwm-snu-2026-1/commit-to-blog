# Workflow 원칙

모든 스킬에 적용되는 협업 원칙이다.

## 핵심 원칙

1. **가정하지 말고 질문한다**
   모호한 상황에서 추측으로 진행하지 않는다.

2. **선택지를 제시할 때는 추천과 이유를 함께 말한다**
   단순히 "A 또는 B?" 가 아니라 "A를 추천합니다. 이유는..." 형식으로.

3. **단계 완료 후 반드시 사용자 승인을 받고 넘어간다**
   각 스킬의 체크포인트를 생략하지 않는다.

4. **모호성 발견 즉시 멈추고 보고한다**
   진행하면서 예상치 못한 문제를 발견하면 즉시 중단하고 알린다.

5. **docs와 rules를 먼저 확인한다**
   새 작업 시작 전, 관련 KB 파일을 먼저 로드한다.

6. **단계 전환 전 반드시 커밋한다**
   각 스킬의 마지막 단계에서 다음으로 넘어가기 전, 변경 사항을 커밋한다.
   커밋 메시지 컨벤션:
   - `workflow/task: [task명]` — whattodo-next 완료 후
   - `workflow/plan: [task명]` — planner 완료 후
   - `workflow/exec: [task명]` — executor 완료 후
   - `workflow/eval: [task명]` — evaluator 완료 후

## 체크포인트 형식

각 스킬에서 사용자 확인이 필요한 시점에는 아래 형식을 쓴다:

```
─────────────────────────
[단계명] 체크포인트
[내용 요약]
─────────────────────────
→ 질문: [구체적인 질문]
```
